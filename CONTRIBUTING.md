# Contributing

Thank you for contributing to `codex-task-contract-skill`.

## Contribution Principles

1. Preserve the core purpose: clarify before action.
2. Keep the Skill concise and practical.
3. Avoid scope creep into a general agent framework.
4. Add examples and fixtures when behavior changes.
5. Keep high-risk actions behind a Risk Gate.

## Development Workflow

1. Create a feature branch.
2. Update canonical files under `skills/task-contract/`.
3. Update docs and tests when behavior changes.
4. Run validation scripts when available.
5. Sync the plugin package before release.

## Pull Request Checklist

- [ ] Behavior change is documented.
- [ ] `SKILL.md` remains concise.
- [ ] Relevant references or assets are updated.
- [ ] Fixtures are added or updated.
- [ ] Risk Gate behavior is preserved.
- [ ] No private chain-of-thought is requested or exposed.
