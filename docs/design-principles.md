# Design Principles

## 1. Clarify Before Action

Codex should first establish what it is doing and how success will be judged.

## 2. Preserve Intent

The Optimized Task must not change the user's goal.

## 3. Bound Scope

The Skill should prevent vague requests from expanding into unrelated work.

## 4. Prefer Decision UX

When a choice matters, present numbered options and recommend a default.

## 5. Make Output Explicit

Every non-trivial task should include an Output Contract.

## 6. Apply Risk Gates

High-impact work should pause for user approval before execution.

## 7. Keep Simple Tasks Simple

Use Compact Contract for small tasks.

## 8. Bound Loops

Loop Contract Mode must include observation, validation, stop conditions, and iteration limits.
