# v0.2.0 Release Checklist

Use this checklist before creating the `v0.2.0` tag.

## Repository

- [ ] README targets v0.2.0.
- [ ] CHANGELOG includes v0.2.0 release-candidate notes.
- [ ] Roadmap marks v0.2.0 as Stable Loop Contract Mode.

## Skill

- [ ] `skills/task-contract/SKILL.md` treats Loop Contract Mode as stable.
- [ ] Compact, Full, and Loop modes remain present.
- [ ] Loop state model is documented.
- [ ] No hidden reasoning is requested.
- [ ] No open-ended loop behavior is encouraged.

## Loop Policy

- [ ] Loop policy is expanded.
- [ ] Observation methods reference exists.
- [ ] Stop conditions reference exists.
- [ ] Escalation rules reference exists.
- [ ] Evaluation rubric exists.

## Templates

- [ ] Full Loop Contract template exists.
- [ ] Compact Loop Contract template exists.
- [ ] Loop Log template includes validation and stop summary.

## Fixtures

- [ ] Loop debug expected output is expanded.
- [ ] Loop research expected output is expanded.
- [ ] Loop documentation expected output is expanded.
- [ ] High-impact loop expected output requires Approval Gate.
- [ ] Repo maintenance loop fixture and expected output exist.

## Validation Scripts

- [ ] `scripts/validate-loop-contract-fixtures.py` exists.
- [ ] `scripts/validate-loop-contract-fixtures.sh` exists.
- [ ] `scripts/validate-repo.sh` calls the loop validator.

## Plugin Package

- [ ] Plugin manifest version is `0.2.0`.
- [ ] Plugin skills path is `./skills/`.
- [ ] Plugin package is synced after canonical Skill changes.

## Validation Commands

```bash
bash scripts/validate-repo.sh
bash scripts/validate-loop-contract-fixtures.sh
bash scripts/sync-plugin-package.sh
bash scripts/validate-repo.sh
git status
```

## Release Commands

```bash
git tag v0.2.0
git push origin v0.2.0
```
