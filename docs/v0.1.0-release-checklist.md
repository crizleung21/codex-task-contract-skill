# v0.1.0 Release Checklist

Use this checklist before creating the `v0.1.0` tag.

## Repository

- [ ] README explains the project in under 60 seconds.
- [ ] LICENSE is present.
- [ ] CHANGELOG includes v0.1.0 notes.
- [ ] CONTRIBUTING and SECURITY docs are present.
- [ ] Repository structure matches `IMPLEMENTATION__PLAN.md`.

## Skill

- [ ] `skills/task-contract/SKILL.md` exists.
- [ ] Skill frontmatter includes `name: task-contract`.
- [ ] Skill description explains compact, full, and loop-aware contracts.
- [ ] Auto-Skeleton fields are documented.
- [ ] Task Optimizer rules are documented.
- [ ] Output Contract is documented.
- [ ] Decision Points are documented.
- [ ] Approval Gate behavior is documented.
- [ ] No hidden reasoning is requested.

## Plugin

- [ ] `plugin/codex-task-contract-skill/.codex-plugin/plugin.json` exists.
- [ ] Plugin version is `0.1.0`.
- [ ] Plugin `skills` path is correct.
- [ ] Packaged Skill copy exists.
- [ ] Local marketplace metadata exists.

## Fixtures

- [ ] Compact Contract fixture exists.
- [ ] Full Contract fixture exists.
- [ ] High-impact fixture exists.
- [ ] Documentation fixture exists.
- [ ] Research fixture exists.
- [ ] Loop-aware fixtures exist as preview coverage.

## Validation Commands

```bash
bash scripts/validate-repo.sh
bash scripts/sync-plugin-package.sh
bash scripts/validate-repo.sh
git status
```

## Release Commands

Run only after review:

```bash
git tag v0.1.0
git push origin v0.1.0
```

## Known v0.1.0 Limitation

Loop Contract Mode is included as a documented preview pattern. v0.2.0 should expand loop golden outputs and validation depth.
