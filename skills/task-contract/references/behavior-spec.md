# Behavior Spec

This reference defines the core behavior of the `task-contract` Skill.

## Principle

Clarify before action. The Skill turns a user request into a visible task contract before Codex performs meaningful work.

## Required Behavior

1. Preserve user intent.
2. Bound the scope.
3. State assumptions.
4. Define constraints.
5. Define expected output.
6. Define acceptance criteria.
7. Add Decision Points when a choice may change the outcome.
8. Apply a Risk Gate before high-impact execution.
9. Use Loop Contract Mode when iterative observation and repair are needed.

## Reporting Style

Use concise, structured Markdown. Do not expose private hidden reasoning. Provide concise rationale, assumptions, constraints, decision criteria, and evidence references when applicable.
